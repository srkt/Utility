# Forecast Module – Cost Forecasting Application

## Overview

The Forecast Module is a cost forecasting application used to collect, manage, and report monthly financial forecast data. Users submit structured monthly updates, which are validated, stored, and used in monthly financial reporting and variance analysis.

The module ensures forecast consistency, historical tracking, and reporting accuracy across financial cycles.

---

## Purpose

This module is designed to:

- Capture monthly cost forecasts from users  
- Maintain historical forecast revisions  
- Support month-over-month variance analysis  
- Provide structured, reporting-ready financial data  
- Ensure traceability and audit readiness  

It acts as the system of record for forecast data used in recurring financial reports.

---

## Technology Stack

- Backend Framework: ASP.NET MVC 5  
- API Layer: ASP.NET Web API  
- Server-Side Language: C#  
- Frontend Framework: Vue.js  
- Frontend Languages: TypeScript, JavaScript  
- Framework Version: .NET Framework 4.7  
- Database: Oracle (with backup configuration enabled)  

---

## How It Works

Each month, users enter updated cost forecast data through the web interface.

1. Data is entered via Vue.js components.
2. Client-side validation is handled using JavaScript/TypeScript.
3. Data is submitted to REST endpoints built with ASP.NET Web API.
4. Server-side validation and business rules are enforced in C#.
5. Validated data is stored in the Oracle database.
6. Stored forecast data is used in monthly reports and financial dashboards.

Historical records are preserved to allow comparison of prior and current forecast submissions.
